# PsiLabs Docs

Project documentation lives here so app source files stay focused on application code.

## Documentation Map

- `AGENT_RULEBOOK.md`: stable coding rules and behavioral instructions for AI agents.
- `CURRENT_TASKS.md`: active implementation tasks, next steps, and acceptance criteria.
- `ARCHIVED_TASKS.md`: completed milestones moved out of current tasks after completion.
- `SCHEMA_NOTES.md`: schema mappings, aliases, defaults, backfillers, field naming, analytics rules, and unresolved schema decisions.
- `ROADMAP.md`: future product direction and long-term architecture ideas.
- `SHARED_VOICE_ENGINE.md`: Tauri-first, platform-agnostic Shared Voice Engine architecture and implementation sequencing.
- `UI_SPACING_AND_LAYOUT.md`: UI spacing, alignment, hierarchy, and layout rules.

## Task Flow

Use this lightweight flow:

1. Capture larger ideas and future bets in `ROADMAP.md`.
2. Promote near-term work into `CURRENT_TASKS.md` with clear acceptance criteria.
3. Move verified completed work into `ARCHIVED_TASKS.md`.

This keeps current work short and actionable while preserving the reasoning behind completed milestones.
